Command Line

Learn Enough Command Line to Be Dangerous, Lesson the First

Getting used to the basics of the command line is somewhat essential to everything else. I've been told we can do pretty much absolutely anything from the command line, but I'm certainly not at that stage - and even illustrious programmers who have a decade of experience have a cheat sheet (I've seen it with mine own jellied orbs).
So what do I know?

  • How to use the terminal to create files and directories
  • How to move around those files and directories
  • How to delete/alter files and directories
  • How to find out about different commands (man pages)
  • How to download files (curl) from the webs
  • Running GREP and the like

Things that I very much need to keep going over because they haven't quite sunk in...

  • Piping things | (less/more, other abstractions...)
  • Running grep searches, they are pesky, and I haven't used it enough to be comfortable

I mean, in general I'm very comfortable using the command line for basic git use - and I think that's because it is what I've used it for most.
Actions going forward are to try and actively move around and use the command line to further my comfortability with it. This will likely be done as I move along the Ruby on Rails Tutorial.

Something I very much do enjoy about the command line is the ability to make it look like I'm in the playground of the matrix, and people look at me like I'm a wizard on the bus...
